# 代码规范
采用 TypeScript 原生 ESM 严格模式；杜绝任意类型，优先使用精准类型、未知类型与窄范围适配封装。禁止添加全局忽略类型校验注解；代码屏蔽注解必须附带合理说明。
分支判断优先采用可区分联合类型、固定编码，摒弃无约束字符串判定；不使用空值、空对象等模糊占位标识。
代码排版适配格式化工具，复杂表达式拆分为布尔变量、数据结构体、小型工具函数，不强制压缩代码格式。
业务逻辑分层解耦：复杂决策逻辑上层处理，函数入参、对象字段仅使用常量、变量读取等简单取值。
多用提前返回写法，减少多层嵌套判断；代码拆解为数据收集→标准化处理→逻辑判定→业务执行四段式结构。
仅为表意清晰定义中间变量，不堆砌无效临时变量。
同一业务模块禁止同时混用静态引入与动态引入，拆分懒加载边界文件；代码修改后编译校验动态引入有效性。
严控模块循环依赖，保持依赖检测工具校验结果合规。
类编写禁止篡改原型属性，采用继承、组合模式设计；若有单元测试，优先实例模拟方式。
单文件代码量接近 700 行时，为提升可读性与可测性合理拆分文件。
