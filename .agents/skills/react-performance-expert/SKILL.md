# React 性能优化专家

构建高性能 React 应用的高级策略。

## 何时应用
- 处理 UI 延迟或交互缓慢时。
- 优化大型列表或复杂数据可视化时。
- 减少包体积和初始加载时间时。
- 分析并修复不必要的重复渲染时。

## 核心规则

### 1. 渲染优化
- **智能记忆化**：仅对重渲染开销大且 Props 频繁以相同值变化的组件使用 `React.memo`。
- **稳定引用**：对传递给记忆化组件的对象/函数使用 `useCallback` 和 `useMemo`。
- **状态局部化**：将状态尽可能保持在靠近使用它的地方，以限制重渲染的范围。

### 2. 加载策略
- **代码分割**：对路由组件和大型第三方库使用 `React.lazy` 和 `Suspense`。
- **图像优化**：确保图像是响应式的且使用了懒加载。

### 3. 大数据处理
- **虚拟化**：对于超过 100 项的列表，使用窗口化/虚拟化技术（如 `react-window`）。
- **避免内联函数**：如果函数被传递给子组件，不要在渲染体内部定义它。

### 4. React 19 并发特性
- **Transitions**：使用 `startTransition` 处理非紧急更新（如过滤列表），以保持 UI 响应。
- **Suspense 集成**：利用 Suspense 进行数据获取，避免“瀑布流”式加载。
