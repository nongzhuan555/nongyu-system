# 农屿 App 登录登出需求规格说明书 (PRD)

## 1. 文档信息

- **版本**: v1.0.0
- **状态**: 正式发布
- **作者**: AI Assistant (Refined from draft)
- **日期**: 2026-06-01
- **描述**: 本文档定义了农屿 App 的用户身份验证流程，包括基于川农教务系统的登录、自动注册、单设备限制及登出逻辑。

***

## 2. 项目背景与目标

### 2.1 背景

农屿 App 旨在为四川农业大学学生提供便捷的服务，为了降低使用门槛并确保用户身份的真实性，农屿采用“川农教务系统直连”模式，跳过传统的注册流程，实现“登录即注册”。

### 2.2 目标

- 实现安全、快速的身份验证。
- 自动化用户档案建立。
- 确保用户数据的一致性与隐私安全（数据库不存储教务密码）。
- 防止多设备同时登录导致的数据冲突。

***

## 3. 业务流程

### 3.1 登录与注册主流程

1. 用户在 App 登录页输入学号（9位数字）和教务系统密码。
2. App端直接调用教务网接口进行身份验证。
3. **验证失败**: 返回错误信息（如学号密码错误等等）。
4. **验证成功**:
   - 后端检查数据库是否存在该学号记录。
   - **新用户**: 自动从教务系统抓取基本信息（姓名、专业、学院、班级等），创建用户记录。
   - **老用户**: 更新“最近使用时间”和“设备信息”。（最近使用时间主要来自于用户每次打开App的埋点采集，登录后触发一次埋点即可，无需在登录流程中增加额外步骤。）
5. 后端生成 Token 返回给 App，App 缓存学号及 Token，跳转至首页。

***

## 4. 功能性需求

### 4.1 登录功能

- **登录凭证**: 四川农业大学本科生教务系统账号（学号）及密码。
- **学号格式**: 固定 9 位数字。
- **密码处理**:
  - **本地端**: 本地缓存（用于自动登录）
  - **远程端**: 后端数据库不存教务网密码，因为教务密码不由农屿系统维护，存的密码易过期无意义且存在安全风险。

### 4.2 自动注册与信息同步

登录成功后，系统需自动维护以下用户信息：

#### 4.2.1 基础信息（自动抓取）

| 字段  | 说明         | 来源        |
| :-- | :--------- | :-------- |
| 学号  | 唯一标识，9位数字  | 用户输入      |
| 姓名  | 用户真实姓名     | 教务系统      |
| 专业  | 如：物联网      | 教务系统      |
| 学院  | 如：信息工程学院   | 教务系统      |
| 班级  | 如：物联网2301班 | 教务系统      |
| 年级  | 如：2023级    | 教务系统      |
| 性别  | 男/女        | 教务系统/二课系统 |
| 生源地 | 如：重庆       | 教务系统      |

#### 4.2.2 扩展信息（系统生成/用户维护）

- **QQ号**: 用户首次登录后可自行填写/修改。
- **设备信息**: 记录当前登录设备的型号、系统版本（如 Android 13.0.0）。
- **时间戳**: 记录 `账号创建时间` 和 `最近登录时间`。
- **在线状态**: 实时/半实时维护（1-在线，0-离线）。
- **数据缓存**: 本地以 JSON 格式缓存课程信息和用户偏好设置。

### 4.3 登出功能

- **操作逻辑**: 用户主动触发登出。
- **本地行为**: 清空本地所有敏感缓存（Token、课程缓存、个人隐私数据），重置为未登录状态，跳转至登录页。
- **远程行为**: 更新在线状态为离线，Token 失效。数据库内用户记录保持不变。

### 4.4 多端登录限制（单设备原则）

- **规则**: 一个学号在同一时间仅允许在一台设备上处于登录状态。
- **逻辑**: 当新设备登录成功时，若旧设备仍有活跃 Session，需强制旧设备下线并提示：“您的账号已在另一台设备登录，请重新登录。”

## 5. UI/UX 设计要求

### 5.1 视觉风格

- **背景**: 采用浅绿色（农屿品牌色）动态渐变效果，营造清新、生机的氛围。
- **布局**: 简约居中布局。

### 5.2 交互元素

- **问候语**: 动态显示（如“欢迎来到农屿”或“农屿，与你在一起”）。
- **输入框**:
  - **学号**: 仅限数字输入，长度限制 9 位。失去焦点时进行正则校验。
  - **密码**: 默认隐藏，提供“显示/隐藏”切换开关。
- **登录按钮**:
  - 初始状态为禁用，当学号与密码均非空且格式正确时激活。
  - 点击后显示 Loading 状态，防止重复提交。
- **帮助信息**: 在底部提供“无法登录？”链接，点击弹出包含“找回教务密码”、“联系客服”等指引。

***

## 6. 异常流程处理

| 场景     | 预期行为                         |
| :----- | :--------------------------- |
| 网络异常   | 弹出 Toast 提示：“网络连接不稳定，请稍后再试”。 |
| 教务系统维护 | 提示：“川农教务系统正在维护中，暂时无法验证登录”。   |
| 凭证错误   | 提示：“学号或密码错误，请检查后重试”。         |
| 格式校验失败 | 输入框下方红字显示：“请输入正确的9位学号”。      |
| 登录超时   | 登录请求超过 15s 未响应，自动重试一次或提示超时。  |

***

## 7. 非功能性需求

- **安全性**: 所有登录请求必须通过 HTTPS 传输。敏感字段（如密码）在传输过程中需加密。
- **性能**: 登录验证响应时间（含后端代理教务系统请求）应控制在 3s 以内。
- **可用性**: 登录页面需适配各种主流手机屏幕尺寸。

***

## 8. 验收标准 (Acceptance Criteria)

1. [ ] 输入合法的 9 位学号和正确密码，能成功进入首页并看到个人姓名。
2. [ ] 首次登录的用户，在数据库管理后台能看到自动生成的完整档案。
3. [ ] 输入 8 位学号，登录按钮不应被激活或点击后提示格式错误。
4. [ ] 在 A 设备登录后，B 设备再次登录，A 设备需被强制弹出至登录页。
5. [ ] 点击“登出”后，杀掉 App 重新进入，应停留在登录页而非首页。
